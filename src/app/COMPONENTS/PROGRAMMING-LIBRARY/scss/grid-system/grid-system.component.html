The grid system allows you to responsively position HTML elements in the dom, independent of their HTML structure.

In the following example, the standard HTML dom order is how your HTML elements are normally displayed by default. They are layed out one after the other, top to bottom.

If these elements are displayed inline, they can then be positioned in the same row; however, they are still forced to maintain their original order from left to right.

Normal positioning techniques allow you to position elements within the above ristrictions.

What the Grid System allows you to do, is no matter the order of the elements, each element can be moved and positioned anywhere in the document tree, regardless of the original HTML order.

If you are familiar with the bootstrap way of responsive positioning through the use of columns ( found here ), the Grid System takes it a step further, allowing you to customize the quantity of columns and how many of them each element should span.

Unlike bootstrap columns, which are declared on the HTML element itself, CSS Grid is applied entirely in CSS. It begins with the parent container being set to "display: grid". in our example above, this parent container is called "wrapper". Simply setting this to display: grid sets it up to receive grid specifications. The first of which, is the"grid-template-columns" "grid-template-columns" specification, which tells the grid how many columns it will be working with.


In the above example, we jump straight into the more elegant shorthand for specifying a number of columns. We use the repeat() method which takes in two arguments; the first argument is the number of columns we would like our grid to have. Bootstrap along with most other column based systems cap this value at 12. With CSS Grid, you can specify whatever number you would like. We will say that we would like to have 24 columns (without the repeat method, you would have to write the second argument 24 separate times).

The second argument is the size. CSS Grid introduces a new value we can make use of. The fraction value. In essence, we are stating that we want 24 columns, each equating to 1 equal part, or fraction, of whatever the total width of the grid is. It is a way to ensure your columns take up exactly 1/24th of the grid and that they size accordingly when the viewport is changed. repeat(24, 1fr)

We can see from the following screenshot that the grid now spans 24 columns and that each element is by default placed within one column.


To tell the grid how many columns each element should span, we can make use of the "grid-column" style property

In the following CSS, some height has been added to the elements in order to better visualize what is going on.

Each element now has the grid-column style which takes in 2 values separated by a forward slash.

{ grid-column: 1 / span 24 }

The first value is the column you wish the element to begin on. Since we want the header to span the entire width of the page, we tell it to start at column 1, and then specify how many columns it should span using span 24.
