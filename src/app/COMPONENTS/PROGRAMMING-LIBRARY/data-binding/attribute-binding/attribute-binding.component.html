ATTRIBUTE BINDING

In order to understand Attribute Binding, it is important to understand the difference between attributes and properties.

Attributes are name value pair statements that are inserted in the start tag of an HTML element in the same way that you would declare a class for that element. An example is the “href” attribute.


<a href="https://www.reddit.com">This is a link</a>

The name in this case is href and the value is the https://www.reddit.com url.

When the Document Object Model (DOM) is rendered, every html element becomes an object. Objects have access to an array of properties that can be dot-walked. The attributes from the HTML initialize a corresponding object property or DOM “node” with the attribute’s value being set as the property value.

Once attributes initialize these properties and their corresponding values, their task is complete; the attributes themselves no longer get changed. Any dynamic changes now happen on the DOM property value.

To visualize this concept, we can look at the “input” element:


<input type="text" value="grapefruit" />

There are two attributes here, but we will focus on the second. This attribute name “value” has an actual content value of “grapefruit”. In this example, the value attribute will create a corresponding DOM node with a value initialized to "grapefruit". When the user types inside the input box, the value they type replaces the current property value. The attribute value on the other hand, always remains grapefruit.

To show this, you can call the attribute by typing: input.getAttribute('value'). This will return “grapefruit”.

On the other hand, you can call the property value by typing: input.value, and it will return whatever the user typed into the input field. If nothing new was typed, it will return the original value, or "grapefruit".

In other words, the HTML attribute value is the initial value while the DOM value property is the current value.

It is important to note the following:

  • There are some HTML attributes that do not generate corresponding properties and thus do not have a 1:1 mapping. “colspan” is one such example. Colspan is an attribute of the DOM, but it does not have a corresponding property. It is read-only yet the browser still renders it because it is an attribute.
  • "Id" on the other hand is an example of 1:1 mapping. If you call document.getElementById(“idname”).id it would return idname.
  • Some DOM properties do not have corresponding attributes. "textContent" is one example.

The disabled attribute is a unique example. The disabled property on a button is set to false by default, meaning the button is enabled by default. If you add the word “disabled” to the starting tag of the button, it becomes disabled. It does not take a value and is declared with the single name alone. Which is why adding disabled="false" does nothing as it does not get registered by the DOM.

To recap, you would only bind to an attribute if it does not have a corresponding property . Attribute binding is an expensive operation because the entire element needs to be re-rendered as opposed to just the property value. Thus it's usage should be avoided if at all possible.
